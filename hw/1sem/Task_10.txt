Задание 10. Разработка системы по технологии разработки через тестирование.
-------------------------------------------------------------------------------
1. 	Создайте в своем рабочем репозитории проект task_10
2. 	Пользуясь технологией разработки через тестирование, реализуйте систему 
	учета платежей по договорам, удовлетворяющую следующим требованиям 
	(можно реализовать не все, но балл будет меньше):
	- 	добавление договора с заданным номером (строка) и датой
		заключения (строка формата "YYYYMMDD", где Y - год, M - месяц, 
		D - день).
	- 	возможность регистрировать платежные документы типа 
		"Платежное поручение" и "Банковский ордер"
	-	добавление платежного докуменда с указанной суммой, номером, типом
		документа, номером договора и датой платежа. 
		Сумма задачется целым положительным числом копеек;
		номер платежного документа - положительное целое число;
		дата платежа - строка формата "YYYYMMDD";
		тип документа задается перечислением.
	-	поиск всех платежей по договору с заданным номером
	-	вычисление суммы всех платежей по договору с заданным номером
	-	уделение платежей с заданным номером, номером договора и датой
	-	получение списка всех договоров с их суммарными суммами платежей
	-	получение списка всех платежей
	--- ОШИБКИ
	-	попытка добавить договор с уже существующим номером - ошибка
	-	попытка добавить договор с пустым номером - ошибка
	-	попытка удалить несуществующий платеж - ошибка
	-	попытка добавить платеж с уже существующим номером договора,
		номером документа, типом документа и датой - ошибка
	-	null в качестве аргументов - ошибка
	-	отрицательная или нулевая сумма платежа - ошибка
	-	обработка ошибок осуществляется с помощью выбрасывания исключений
		с соответствующим сообщением
2.1	Если какие-то требования нуждаются в уточнении - пишите мне, уточню. 
	Это нормальная практика работы заказчика и разработчика.
3.	Реализуйте консольную утилиту для работы с системой, поддерживающую команды 
	для всех реализованных фич. 
	При запуске утилиты пользователь должен видеть меню с возможностью выбрать
	команду вводом ее номера.
	Если в процессе работы возникает ошибка, соответствующее сообщение выводится
	на экран, после чего работа продолжается.
4.	Для получения полного балла важно соблюдать порядк разработки и своевременно 
	делать коммиты изменений. Подробнее об этом говорилось на занятии.
5.	Создайте в проекте текстовый файл со списком реализованных фич с именем 
	features.txt
6.	Загрузите изменения в репозиторий не позднее 11.01.22, 23:59:00.999
